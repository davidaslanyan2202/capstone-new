# File Manifest

This manifest classifies the repository after the IEEE-oriented cleanup.

| Category | Paths | Notes |
| --- | --- | --- |
| Code | `code/preprocessing/`, `code/analysis/`, `code/scraping/`, `code/validation/`, `code/reproduce.py` | Project scripts for preprocessing, scraping, modeling, validation, and one-command reproduction. |
| Raw data | `data/raw/` | Original CSV inputs used to build the project dataset. |
| Scraped data | `data/scraped/transfermarkt/history_json/` | Transfermarkt transfer-history JSON files, one file per player ID. |
| Inferred/interim data | `data/interim/` | Derived CSVs such as ID-enriched metrics, valuation dates, transfer events, market values, contract features, and Transfermarkt ID review files. |
| Processed data | `data/processed/` | Final analytics and modeling datasets used by the analysis workflow. |
| Generated reports | `reports/generated/` | Programmatically generated figures, tables, and Markdown analysis outputs. |
| Authored reports | `reports/authored/` | Hand/AI-written presentation, poster, and Q&A materials that are not regenerated by code. |
| IEEE final paper | `paper/final/` | Canonical IEEE paper source, bibliography, figures, and compiled PDF. Reproduction refreshes only `paper/final/figures/`. |
| Project docs | `docs/project/` | Proposal, data plan, and implementation plan notes. |
| Guidelines | `docs/guidelines/` | Capstone paper/project and poster guideline PDFs. |

## Reproduction Contract

Use `python code/reproduce.py` to rebuild the main non-network outputs from tracked data. The workflow uses tracked scraped JSON files and does not call the scraper by default.

Reproduction now runs validation gates after ID assignment and analytics-table construction. Ambiguous exact-name Transfermarkt matches are excluded from `metrics.csv` and written to `data/interim/ambiguous_transfermarkt_id_rows.csv`; unmatched rows are written to `data/interim/unmatched_transfermarkt_id_rows.csv`.

The scraper in `code/scraping/scrape_transfer_history.py` is optional and network-dependent. Run it only when the scraped Transfermarkt history JSON files need to be refreshed.

Assistant-internal artifacts such as graph analysis caches are intentionally excluded from the submission archive.
